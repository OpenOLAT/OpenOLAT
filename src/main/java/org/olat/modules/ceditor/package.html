Overview

The ceditor package implements a server-centric, component-based content editor for OpenOlat. Pages consist of ordered PagePart elements stored via single-table JPA inheritance. Each element type is managed by a PageElementHandler that provides view, editor, and inspector controllers.

Usage Contexts

  • Portfolio pages — via PageRunController in org.olat.modules.portfolio.ui
  • Evaluation forms — via EvaluationFormEditorController in org.olat.modules.forms.ui

Key Principle: All state lives on the server. The browser receives HTML fragments via AJAX. There is no client-side framework — drag-and-drop uses the lightweight dragula.js library.

AbstractPart — Single-Table Inheritance

@Entity(name="cepagepart")
@Table(name="o_ce_page_part")
@DiscriminatorColumn
public abstract class AbstractPart implements PagePart {
    @Id @GeneratedValue(strategy = IDENTITY)
    private Long key;

    @Column(name="p_content")    // HTML or structured data
    private String content;

    @Column(name="p_layout_options") // XStream XML settings
    private String layoutOptions;

    @Column(name="pos", updatable=false) // ordering (read-only)
    private long pos;

    @ManyToOne(targetEntity=PageBodyImpl.class)
    @JoinColumn(name="fk_page_body_id")
    private PageBody body;
}

Settings Serialization

ContentEditorXStream provides a secure XStream instance with explicit type permissions. It implements the dual storage pattern:

  • content column — HTML text or structured data (e.g. TableContent as XML)
  • layoutOptions column — XStream XML for settings objects

public class TitlePageElementHandler implements
        PageElementHandler, PageElementStore<TitleElement>,
        SimpleAddPageElementHandler, CloneElementHandler {

    @Override
    public PageElement createPageElement(Locale locale) {
        TitlePart title = new TitlePart();
        TitleSettings settings = new TitleSettings();
        settings.setSize(3);
        settings.setLayoutSettings(BlockLayoutSettings.getPredefined());
        title.setTitleSettings(settings);
        return title;
    }

    @Override
    public PageRunElement getContent(...) {
        String htmlContent = TitleElement.toHtml(content, settings);
        TextComponent cmp = TextFactory.createTextComponentFromString(...);
        return new TextRunComponent(cmp, false);
    }
}

Markdown Import

The markdown import feature converts CommonMark markdown into page content. Users can upload .md/.txt files, .zip archives (containing a single .md with images), or paste markdown text directly.

Conversion pipeline: MarkdownMathPreprocessor → CommonMark parser (+ GFM Tables extension) → MarkdownPagePartVisitor → persist parts → wrap in ContainerPart (block_1col).

Security: Inline HTML is entity-escaped (escapeHtml(true)), HTML blocks are skipped, URL sanitization allows only http/https/mailto protocols, remote image downloads are restricted to allowed domains (MediaServerModule), and local file paths are blocked in text paste mode.

UI: MarkdownImportController (FormBasicController) with file upload or text paste tabs. Enabled when PageEditorProvider.isImportMarkdownEnabled() returns true.

Element Type Reference

Creating Elements Programmatically

Title

TitlePart title = new TitlePart();
title.setContent("My Heading");
TitleSettings ts = new TitleSettings();
ts.setSize(2); // h2
ts.setLayoutSettings(BlockLayoutSettings.getPredefined());
title.setTitleSettings(ts);
pageService.appendNewPagePart(page, title);

Paragraph

ParagraphPart p = new ParagraphPart();
p.setContent("<p>Text with <strong>bold</strong> and <em>italic</em></p>");
pageService.appendNewPagePart(page, p);

Table

TablePart tablePart = new TablePart();
TableContent tc = new TableContent(3, 2);
tc.addContent(0, 0, "Header 1");
tc.addContent(0, 1, "Header 2");
tc.addContent(1, 0, "Cell A");
tc.addContent(1, 1, "Cell B");
tablePart.setContent(ContentEditorXStream.toXml(tc));
TableSettings settings = new TableSettings();
settings.setColumnHeaders(true);
settings.setBordered(true);
tablePart.setLayoutOptions(ContentEditorXStream.toXml(settings));
pageService.appendNewPagePart(page, tablePart);

Code

CodePart codePart = new CodePart();
codePart.setContent("public class Hello { }");
CodeSettings cs = new CodeSettings();
cs.setCodeLanguage(CodeLanguage.java);
codePart.setLayoutOptions(ContentEditorXStream.toXml(cs));
pageService.appendNewPagePart(page, codePart);

Image (via Media Center)

Media media = imageHandler.createMedia(
    "photo", "description", "alt text",
    file, "photo.png", null, author, MediaLog.Action.UPLOAD);
MediaPart mediaPart = MediaPart.valueOf(author, media);
pageService.appendNewPagePart(page, mediaPart);

Adding a New Element Type

  1. Model interface in model/ — extend PageElement or suitable sub-interface
  2. JPA entity in model/jpa/ — extend AbstractPart, override getType(), implement copy()
  3. Register in persistence.xml — add <class> entry
  4. Settings class in model/ (if needed) — register in ContentEditorXStream
  5. Handler in handler/ — implement PageElementHandler + appropriate mixins
  6. Editor controller in ui/ — implement PageElementEditorController
  7. Inspector controller in ui/ — extend PageElementInspectorController
  8. Run component in ui/ — implement PageRunElement
  9. i18n keys in ui/_i18n/
  10. Register handler in provider implementations (PageRunController, EvaluationFormEditorController)
Tip: Use TitlePageElementHandler as the canonical reference implementation. It demonstrates the full handler pattern including SimpleAddPageElementHandler, CloneElementHandler, and PageElementStore<TitleElement>.

Client-Side Integration

The editor relies on two JavaScript libraries:

  • jquery.contenteditor.v3.js — jQuery plugin for element selection, AJAX command dispatch
  • dragula.js — Lightweight drag-and-drop library for element reordering

Commands dispatched from browser via AJAX: edit_fragment, save_element, clone_element, delete_element, move_up, move_down, drop_fragment, add_element_above, add_element_below, add_to_container.
